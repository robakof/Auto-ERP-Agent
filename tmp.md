Używanie `echo "..." > plik` jako fallback do zapisu pliku tymczasowego nadal wymaga zatwierdzenia od użytkownika — tak samo jak każda inna komenda Bash. Właściwy wzorzec to Write tool (po wcześniejszym odczycie pliku). Problem pojawia się gdy plik nie był wcześniej odczytany w sesji — agent sięga po echo zamiast najpierw odczytać plik. Propozycja: dodać do DEVELOPER.md regułę "przed użyciem Write odczytaj plik jeśli istnieje; jeśli nie istnieje — użyj Write bezpośrednio (nie echo)".
